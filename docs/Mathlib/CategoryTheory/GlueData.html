    Gluing data #

    We define GlueData as a family of data needed to glue topological spaces, schemes, etc. We provide the API to realize it as a multispan diagram, and also states lemmas about its interaction with a functor that preserves certain pullbacks.

    source
    structure CategoryTheory.GlueData (C : Type u₁) [CategoryTheory.Category.{v, u₁} C] :
    Type (max u₁ (v + 1))

    A gluing datum consists of

    1. An index type J
    2. An object U i for each i : J.
    3. An object V i j for each i j : J.
    4. A monomorphism f i j : V i j ⟶ U i for each i j : J.
    5. A transition map t i j : V i j ⟶ V j i for each i j : J. such that
    6. f i i is an isomorphism.
    7. t i i is the identity.
    8. The pullback for f i j and f i k exists.
    9. V i j ×[U i] V i k ⟶ V i j ⟶ V j i factors through V j k ×[U j] V j i ⟶ V j i via some t' : V i j ×[U i] V i k ⟶ V j k ×[U j] V j i.
    10. t' i j k ≫ t' j k i ≫ t' k i j = 𝟙 _.
    • J : Type v
    • U : self.J → C
    • V : self.J × self.J → C
    • f : (i j : self.J) → CategoryTheory.GlueData.V self (i, j) ⟶ CategoryTheory.GlueData.U self i
    • f_mono : ∀ (i j : self.J), CategoryTheory.Mono (CategoryTheory.GlueData.f self i j)
    • f_hasPullback : ∀ (i j k : self.J), CategoryTheory.Limits.HasPullback (CategoryTheory.GlueData.f self i j) (CategoryTheory.GlueData.f self i k)
    • f_id : ∀ (i : self.J), CategoryTheory.IsIso (CategoryTheory.GlueData.f self i i)
    • t : (i j : self.J) → CategoryTheory.GlueData.V self (i, j) ⟶ CategoryTheory.GlueData.V self (j, i)
    • t_id : ∀ (i : self.J), CategoryTheory.GlueData.t self i i = CategoryTheory.CategoryStruct.id (CategoryTheory.GlueData.V self (i, i))
    • t' : (i j k : self.J) → CategoryTheory.Limits.pullback (CategoryTheory.GlueData.f self i j) (CategoryTheory.GlueData.f self i k) ⟶ CategoryTheory.Limits.pullback (CategoryTheory.GlueData.f self j k) (CategoryTheory.GlueData.f self j i)
